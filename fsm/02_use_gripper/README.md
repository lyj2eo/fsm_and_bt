# 02. fsmlib 으로 그리퍼 FSM 만들기

## 시나리오
협동로봇 평행 그리퍼는 명령(`close`, `open`)이 들어왔을 때 곧바로 끝나는 게 아니라,
실제 죠(jaw)가 움직이는 동안의 **과도 상태**를 거친다.

```
     ┌──────┐   close   ┌─────────┐ 
     │ Open │ --------→ │ Closing │ 
     └──────┘           └─────────┘ 
      ↑ done               ↓ done
  ┌─────────┐    open   ┌───────┐ 
  │ Opening │ ←-------- │ Close │ 
  └─────────┘           └───────┘ 
```

이 4개 상태를 `fsm::State` 를 상속해 만들고, `fsm::StateMachine` 으로 묶어
시나리오에 따라 전이가 일어나도록 구현한다.

## 작업할 파일
- `main.cpp` — 비어 있는 4개 상태 클래스와 `runScenario()` 의 흐름을 채운다.
- `fsm/fsmlib` 는 **수정하지 않는다.** (라이브러리는 그대로 두고 사용만 한다.)

## 시나리오 / 기대 동작
`main` 의 시뮬레이션은 다음과 같이 흘러야 한다.

1. 초기 상태: `Open`
2. `close` 명령 → `Closing` 으로 전이
3. `Closing` 상태가 N 틱(예: 3 틱) 지속되면 스스로 `Closed` 로 전이
4. `open` 명령 → `Opening`
5. `Opening` 상태가 N 틱 지속되면 스스로 `Open` 으로 전이

각 상태의 `onEnter / onUpdate / onExit` 에서 무엇을 출력할지는 학습자가 정한다.
다만 다음을 만족해야 한다.

- 상태 전이가 일어날 때 어떤 상태에서 어떤 상태로 갔는지 한눈에 보여야 한다.
- "동작 중" 상태(`Closing`, `Opening`)는 진행이 보여야 한다 (예: `[Closing] tick 1/3`).

## 힌트
- 자기 자신을 다음 상태로 전이시키려면 `State` 가 `StateMachine` 에 접근할 수 있어야 한다.
  생성자 인자로 `StateMachine*` 을 받는 게 가장 단순하다.
- 명령(`close`, `open`)은 main 에서 `sm.transitionTo(...)` 를 호출하는 식으로 흉내내면 된다.
- "N 틱 지속" 같은 카운트는 상태 클래스의 멤버 변수로 가지고 있고, `onEnter` 에서 0 으로 초기화한다.
